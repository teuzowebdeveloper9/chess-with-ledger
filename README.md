# Chess With Ledger

[![CI](https://github.com/teuzowebdeveloper9/chess-with-ledger/actions/workflows/ci.yml/badge.svg)](https://github.com/teuzowebdeveloper9/chess-with-ledger/actions/workflows/ci.yml)

## Project Overview

Chess With Ledger is a local multiplayer chess system. Two players share the same screen: one controls white pieces, the other controls black pieces, alternating turns until the match ends.

The main product difference is the immutable Ledger. Every match action creates chronological Ledger events:

- match started
- piece moved
- capture
- check
- castling
- promotion
- board snapshot
- match finished

The admin page at `/admin/ledger` is protected by a password configured in `.env`.

## Stack

- Frontend: React, Vite, TypeScript, Tailwind CSS, `react-icons`, Lucide icons.
- Backend: NestJS, TypeScript, Clean Architecture, DTO validation.
- Chess rules: `chess.js` behind an application port.
- Ledger database: DynamoDB Local through Docker.
- Relational/statistics database: PostgreSQL through Docker.
- Tests: Jest with SWC.
- Infra: Docker Compose.

## Data Ownership

DynamoDB Local stores the immutable Ledger:

- complete move history
- chronological events
- board snapshots
- audit logs

PostgreSQL stores relational and statistical data:

- players
- matches
- duration
- wins/losses/draws
- move count
- scoreboards
- match summaries

## Getting Started

```bash
cp .env.example .env
npm install
npm run docker:up
```

Open:

- Web: `http://localhost:5174`
- API: `http://localhost:3001`
- PostgreSQL: `localhost:5433`
- Admin Ledger: `http://localhost:5174/admin/ledger`

The default admin password is documented in `.env.example`; change it in `.env`.

## API Summary

- `POST /matches`: starts a local match.
- `GET /matches`: lists recent matches.
- `GET /matches/scoreboard`: returns scoreboard data.
- `GET /matches/:matchId`: returns a full match JSON state.
- `POST /matches/:matchId/moves`: records a typed chess move and appends Ledger events.
- `POST /admin/session`: exchanges admin password for a short-lived admin token.
- `GET /admin/ledger`: protected Ledger view endpoint.

## Frontend Notes

The board renders from `BoardState` JSON returned by the backend. React components do not own chess rules.

UI styling uses Tailwind utility classes in JSX. The only frontend CSS file is the Tailwind entrypoint:

```css
@import "tailwindcss";
```

Chess pieces use `react-icons/gi` components. Do not replace them with emoji or Unicode glyph maps.
